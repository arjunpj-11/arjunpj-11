### [Imminiq — Community-Evolving Learning Platform](https://github.com/arjunpj-11/Imminiq)

**AI starts the tracker. The community makes it better.**

</div>

Learning a new subject often begins with hours of searching across websites, videos, documentation, and courses—and important topics can still be missed. Imminiq replaces that fragmented process with a structured, living learning tracker.

AI creates the first version from a learner's goal, level, and preferences. People following the same tracker then form a community: they learn together, discuss ideas, challenge one another, propose missing topics, and verify useful contributions. As the community grows, the tracker evolves with it, giving every future learner a stronger path than the one before.

Imminiq is the project where I bring together this product vision with serious engineering. It is built as a Turborepo monorepo with strict clean architecture, keeping domain logic independent from frameworks and infrastructure.

| What it does | How it is built |
|---|---|
| Creates a structured first tracker from a learner's goal and level | AI workflows processed through BullMQ workers |
| Lets learners publish, discover, review, and clone community trackers | Community modules backed by MongoDB and Redis |
| Evolves trackers through topic proposals and reviewed contributions | Structured contribution and verification workflows |
| Brings people around a tracker to learn and challenge one another | Guilds, shared learning, and real-time experiences |
| Supports lessons with contextual AI guidance and visual explanations | Multiple LLM providers with fallback handling |
| Runs code directly inside the learning experience | Sandboxed execution through Piston |
| Protects accounts and active sessions | OAuth, 2FA, CSRF protection, rate limiting, and refresh-token rotation |
| Keeps the codebase maintainable as it grows | Modular clean architecture with automated architecture checks |

**🛠️ Engineering highlights**

- ✅ 313 automated backend and frontend tests
- ✅ Community publishing, cloning, reviews, contribution requests, and peer verification
- ✅ Real-time features powered by Socket.IO
- ✅ Redis-backed caching, sessions, OTP flows, and background jobs
- ✅ Refresh-token reuse detection and remote session revocation
- ✅ CI with linting, builds, dependency review, and CodeQL scanning

`React` · `TypeScript` · `Node.js` · `Express` · `MongoDB` · `Redis` · `Socket.IO` · `BullMQ` · `Piston` · `Turborepo`
